# Story S04: Featured projects
As a visitor, I want to see a few of Roberto's major projects with brief descriptions, so that I can evaluate the actual work he has done.

## Acceptance Criteria
- Given the home page, when I reach the projects section, then I see 2–4 featured projects.
- Given each featured project, when it is displayed, then it shows a title and a brief description covering what it was and Roberto's role or outcome.

## Non-goals
- Full project case-study pages (V2)
- Surfacing repo links now (deferred to V2 — links attach per-project later)
- Filtering, sorting, or tagging of projects

## Definition of Done
See `_DEFINITION-OF-DONE.md` (shared — applies to all stories).

## Notes (build constraints — not acceptance criteria)
- Design the project card/layout so a repo/proof link can be attached per-project later (V2) without a redesign. This is a forward-compatibility constraint for the builder, not a visitor-observable outcome — hence a note, not an AC.
